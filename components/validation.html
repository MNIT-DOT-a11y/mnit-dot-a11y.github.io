Validation Component

Form validation is the process of ensuring that user input is correct and complete before it is submitted. An accessible validation component provides clear, timely, and programmatically identifiable feedback to all users, helping them correct errors efficiently.

Passed WCAG 2.1 AA

Interactive Example

Form with Validation

This field is required.

Please enter a valid email address.

Source Code

<form id="validation-form" novalidate>
    <div class="form-group">
        <label for="required-field">Required Field</label>
        <input type="text" id="required-field" name="required-field" required>
        <p id="required-field-error" class="validation-message">This field is required.</p>
    </div>
    <div class="form-group">
        <label for="email-field">Email Address</label>
        <input type="email" id="email-field" name="email-field" required>
        <p id="email-field-error" class="validation-message">Please enter a valid email address.</p>
    </div>
    <button type="submit" class="usa-button usa-button--primary">Submit</button>
</form>

Guidance

Accessible form validation is crucial for helping users, especially those with cognitive disabilities or screen readers, understand and correct input errors. Errors should be communicated both visually and programmatically.

When to use the validation component

  • Forms. All forms that require user input should have validation to ensure the data is correct.
  • Required fields. Explicitly indicate which fields are required using a visual marker and a text description.

Usability tips

  • **Clear messaging.** The error message should clearly state what the error is and how to fix it.
  • **Live feedback.** Provide validation feedback in real-time as the user types, where appropriate, to help them correct errors instantly.

Accessibility Features

  • Semantic HTML. The form, label, and input tags provide a solid semantic foundation.
  • Programmatic Association. Use aria-describedby to link the input field to its error message. This allows screen readers to announce the error when the user focuses on the field.
  • Explicit required attribute. The "required" attribute visually and programmatically indicates that a field must be filled out.
  • Visual Indicators. Use a consistent visual style, such as a red border or an icon, to indicate an error state.

Testing

To ensure the validation component meets accessibility standards, refer to the Validation Component Testing Checklist.
