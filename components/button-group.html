Button Group Component

A button group is a set of related buttons displayed together in a single row or column. These are often used for filtering, selecting options, or for groups of actions that are closely related.

Passed WCAG 2.1 AA

Interactive Example

Standard Button Group

Source Code

<h3 class="site-preview-heading">Standard Button Group</h3>
<div class="button-group" role="group" aria-label="Text alignment">
    <button class="button-group-item">Left</button>
    <button class="button-group-item active">Center</button>
    <button class="button-group-item">Right</button>
</div>

<div class="button-group" role="group" aria-label="File options">
    <button class="button-group-item">Save</button>
    <button class="button-group-item">Cancel</button>
    <button class="button-group-item">Delete</button>
</div>

Guidance

Use button groups when you have a set of controls that are related in function and should be displayed as a single, cohesive unit. This improves visual clarity and helps users understand the relationship between the options.

When to use the button group component

  • Toggle options. For selecting a single option from a set, such as a size or alignment.
  • Related actions. For a group of actions that are frequently used together, like Save and Cancel buttons.

Usability tips

  • Clear labels. Each button in the group should have a clear and concise label that describes its purpose.
  • Visible state. The currently active or selected button should have a clear visual style to indicate its state.

Accessibility Features

  • Group Role. Using role="group" on the container helps screen readers understand that the buttons are related.
  • Group Label. The "aria-label" on the container provides a descriptive name for the entire group of buttons.
  • Keyboard Operability. Individual buttons are focusable and can be activated with the Enter or Space key.
  • Current State. The `aria-current="true"` attribute (or an "active" class with equivalent styling) can be used to indicate the selected button.

Testing

To ensure the button group component meets accessibility standards, refer to the Button Group Component Testing Checklist.
