Memorable Date Component

A memorable date component consists of three separate input fields for a date's month, day, and year. This method is often used to make date entry easier for users by breaking the date into smaller, more manageable chunks. It is especially useful for information like a user's date of birth or a key event date.

Passed WCAG 2.1 AA

Interactive Example

Standard Memorable Date Input

Date of birth

Source Code

<fieldset>
    <legend>Date of birth</legend>
    <div>
        <label for="dob-month">Month</label>
        <input type="text" id="dob-month" name="dob-month" placeholder="MM" maxlength="2">
    </div>
    <div>
        <label for="dob-day">Day</label>
        <input type="text" id="dob-day" name="dob-day" placeholder="DD" maxlength="2">
    </div>
    <div>
        <label for="dob-year">Year</label>
        <input type="text" id="dob-year" name="dob-year" placeholder="YYYY" maxlength="4">
    </div>
</fieldset>

Guidance

The memorable date component provides a user-friendly and accessible alternative to a single date input field. By separating the fields, you can reduce cognitive load and simplify data entry, especially for users who may not be familiar with a specific date format.

When to use the memorable date component

  • Birth dates. Use for collecting a person's date of birth in a clear and structured way.
  • Forms with date requirements. Ideal for any form field where a date is a required input, and clarity is prioritized.

Usability tips

  • Clear labels. Each input field should have a label (`Month`, `Day`, `Year`) that is correctly associated with it.
  • Group fields. Use a `
    ` and a `` to visually and semantically group the three input fields together, providing a clear heading for the entire component.

Accessibility Features

  • **Semantic Grouping.** The `
    ` and `` elements are crucial for accessibility, as they provide a single, clear label for the entire group of related inputs. Screen readers will announce the legend ("Date of birth") before announcing each individual field.
  • **Descriptive Labels.** Each input field is correctly associated with its label using `for` and `id` attributes. This ensures screen readers can correctly identify the purpose of each field.
  • **Keyboard Operability.** The component must be fully navigable by keyboard, with a logical tab order that moves from month to day to year.
