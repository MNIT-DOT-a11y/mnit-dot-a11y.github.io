Table Component

A table component is used to organize and display data in a grid with rows and columns. An accessible table is structured in a way that allows screen readers to understand the relationship between headers and their associated data cells, making the data navigable and comprehensible for all users.

Passed WCAG 2.1 AA

Interactive Example

Standard Data Table

Product Sales by Quarter
Product Q1 Sales Q2 Sales Q3 Sales
Laptop $10,000 $12,000 $15,000
Monitor $8,000 $9,500 $11,000
Keyboard $2,500 $3,000 $4,000

Guidance

A data table should be used exclusively for tabular data. Avoid using tables for non-tabular layouts, as this can be confusing for screen reader users and can lead to a poor user experience.

When to use the table component

  • Tabular data. Ideal for displaying spreadsheets, financial reports, or any data with a logical row and column structure.
  • Comparison charts. For comparing features or prices of different products in a clear grid format.

Usability tips

  • Clear headers. Each column and row should have a clear header that describes the data below or beside it.
  • Responsive design. Tables should be designed to be usable on smaller screens, which may involve scrolling or collapsing columns.

Accessibility Features

  • Caption. The element provides a programmatic name for the table, giving screen readers a quick summary of the table's purpose.
  • Table Headers. Using "th" elements with a cope="col" or `cope="row" attribute explicitly associates the header with the corresponding data cells, which is vital for screen readers to correctly announce the data.
  • Semantic Markup. The table, thead, tbod>, tr, and td tags provide a clear semantic structure for the table.
