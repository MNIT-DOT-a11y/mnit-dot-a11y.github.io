Table

Accessible tables are essential for presenting tabular data. They must have a clear structure with proper headers to allow screen readers to understand the relationship between data cells.

Interactive Example

Monthly Sales Report
Month Sales ($) Units Sold
January $50,000 1,200
February $65,000 1,550
March $72,000 1,800

Source Code

<table>
    <caption>Monthly Sales Report</caption>
    <thead>
        <tr>
            <th scope="col">Month</th>
            <th scope="col">Sales ($)</th>
            <th scope="col">Units Sold</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td data-label="Month">January</td>
            <td data-label="Sales">$50,000</td>
            <td data-label="Units Sold">1,200</td>
        </tr>
        <tr>
            <td data-label="Month">February</td>
            <td data-label="Sales">$65,000</td>
            <td data-label="Units Sold">1,550</td>
        </tr>
        <tr>
            <td data-label="Month">March</td>
            <td data-label="Sales">$72,000</td>
            <td data-label="Units Sold">1,800</td>
        </tr>
    </tbody>
</table>

Accessibility Guidance

  • Use the `` element to provide a descriptive title for the table.
  • Use `` for the table headers and `` for the table body.
  • Use `` for all table headers and add a `scope="col"` or `scope="row"` attribute to explicitly define their relationship to the data cells.
  • For responsive design on mobile, use `data-label` attributes on `` elements to show a label when the table collapses.
