Site Alert Component

A site alert is a banner-style message used to communicate important, and often time-sensitive, information to users. It is typically displayed prominently at the top of a page to ensure visibility. Accessible site alerts are designed to be easily perceivable and to convey their urgency without being disruptive to screen reader users.

Passed WCAG 2.1 AA

Interactive Example

Standard Alerts

Informational alert!

This is an informational alert to let you know about a site update.

Warning alert!

A warning has been issued. Please review your settings to avoid issues.

Success!

Your changes have been saved successfully.

Error!

There was an error processing your request. Please try again later.

Source Code

<div class="usa-alert usa-alert--info" role="alert">
    <div class="usa-alert__body">
        <h4 class="usa-alert__heading">Informational alert!</h4>
        <p class="usa-alert__text">This is an informational alert to let you know about a site update.</p>
    </div>
</div>

Guidance

Site alerts should be used to communicate critical information that is relevant to the user's current task or the overall site status. Avoid using alerts for non-critical information to prevent alert fatigue.

When to use a site alert component

  • Critical system messages. For unplanned outages or emergency notices.
  • Status updates. To inform users of a successful action, a warning, or a failure.

Usability tips

  • Clear language. The message should be clear, concise, and easy to understand at a glance.
  • Consistent placement. Alerts should always appear in a consistent location on the page, typically at the top.

Accessibility Features

  • ARIA Live Region. The "role="alert"" attribute is a live region that causes screen readers to immediately announce its content when it appears, making it suitable for urgent information.
  • Semantic Tags. The use of a heading "h4" and paragraph "p" within the alert provides a clear structure for assistive technologies.
  • Sufficient Contrast. The colors for each alert type have been chosen to meet WCAG 2.1 AA contrast standards for both text and non-text elements.

Testing

To ensure the site alert component meets accessibility standards, refer to the Site Alert Component Testing Checklist.
