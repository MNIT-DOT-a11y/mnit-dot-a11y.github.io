Side Alert Component

A side alert is a non-disruptive way to display status messages or important information that doesn't require immediate user action. This type of alert often appears in a fixed position, sliding in from the side or bottom of the screen.

Interactive Example

Your changes have been saved successfully!

Source Code

<!-- Interactive Button -->
<button id="showAlertButton" type="button">Show Alert</button>

<!-- Alert Component -->
<div id="sideAlert" class="side-alert" role="alert" aria-live="polite">
  <span>Your changes have been saved successfully!</span>
  <button class="side-alert-close-btn" aria-label="Close alert">&times;</button>
</div>

<style>
.side-alert {
    position: fixed;
    bottom: 20px;
    right: -350px;
    width: 300px;
    padding: 20px;
    background-color: #d4edda;
    color: #155724;
    border: 1px solid #c3e6cb;
    border-radius: 5px;
    box-shadow: 0 4px 6px rgba(0,0,0,0.1);
    transition: right 0.5s ease-in-out;
    z-index: 1000;
    display: flex;
    justify-content: space-between;
    align-items: center;
}
.side-alert.show {
    right: 20px;
}
.side-alert-close-btn {
    background: none;
    border: none;
    color: #155724;
    font-size: 1.5em;
    cursor: pointer;
}
</style>

<script>
document.getElementById('showAlertButton').addEventListener('click', function() {
    const alert = document.getElementById('sideAlert');
    alert.classList.add('show');
    // Hide the alert after 5 seconds
    setTimeout(() => {
        alert.classList.remove('show');
    }, 5000);
});

document.querySelector('.side-alert-close-btn').addEventListener('click', function() {
    document.getElementById('sideAlert').classList.remove('show');
});
</script>

Accessibility Guidance

  • `aria-live` and `role="alert"`: Use these attributes on the alert container to ensure screen readers announce the content as soon as it appears on the page.
  • Clear Labeling: Provide a clear, concise text message. If there's a close button, it must have an accessible name using `aria-label` to let screen reader users know its purpose.
  • Do Not Steal Focus: Alerts should not take focus away from the user's current position on the page, as this can be disorienting.
  • Keyboard and Mouse Support: Ensure users can dismiss the alert using both the keyboard (e.g., Tab to the close button and press Enter/Space) and mouse.
