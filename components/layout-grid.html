Layout Grid Component

A layout grid helps align and structure content on a page. It uses a system of columns and gutters to create a consistent, orderly, and responsive design that adapts across different screen sizes.

    </div>

        <div class="col-8"><div>8</div></div>
        <div class="col-4"><div>4</div></div>
    </div>
</div>

Guidance

Grids are the foundation of page layout. A consistent grid system improves readability and user experience by creating visual rhythm and predictability.

When to use a layout grid

  • Page structure. To define the main regions of a page, such as headers, footers, sidebars, and main content areas.
  • Content alignment. To align cards, images, text blocks, and other components in a visually pleasing and organized way.
  • Responsive design. To ensure that page layouts adapt gracefully from large desktop monitors to small mobile screens.

Usability tips

  • Choose a standard. A 12-column grid is a flexible and widely-used standard that allows for many column combinations (halves, thirds, quarters).
  • Mind the gap. Use consistent spacing (gutters) between columns to prevent content from running together and to improve readability.
  • Mobile-first. Design your grid to work on small screens first, then add complexity for larger screens. Content should stack vertically in a single column by default.

Accessibility Features

  • Logical source order. The order of elements in the HTML (DOM order) should make sense without CSS. Screen readers follow the DOM order, not the visual order, so content should be structured logically.
  • Semantic containers. Use semantic HTML elements like "main", "nav", "aside", and "section" as grid containers instead of using "div" for everything. This adds meaning and improves page navigation for assistive technology users.
  • Responsiveness. The grid must be responsive to prevent horizontal scrolling on small screens, which is a failure of WCAG 1.4.10 Reflow.
