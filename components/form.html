Form Component

A form is a set of inputs and controls that allows users to submit information. Forms should be designed to be easy to use and navigate for all users, including those who rely on assistive technologies.

Passed WCAG 2.1 AA

Interactive Example

Standard Form

Please enter your name.

Please enter a valid email address.

Source Code

<form id="contact-form" novalidate>
    <div class="form-group">
        <label for="name">Name</label>
        <input type="text" id="name" name="name" required aria-describedby="name-error">
        <p id="name-error" class="error-message">Please enter your name.</p>
    </div>
    <div class="form-group">
        <label for="email">Email</label>
        <input type="email" id="email" name="email" required aria-describedby="email-error">
        <p id="email-error" class="error-message">Please enter a valid email address.</p>
    </div>
    <button type="submit" class="usa-button usa-button--primary">Submit</button>
</form>

Guidance

Forms are often the primary way users interact with a website. An accessible form ensures that all users can complete tasks, regardless of their abilities.

When to use a form component

  • Data collection. Use for collecting user information, such as names, email addresses, or feedback.
  • User actions. Forms are necessary for actions like logging in, signing up, or making a purchase.

Usability tips

  • Clear labels. All form controls must have a visible and correctly associated label.
  • Logical tab order. Ensure the keyboard focus moves in a logical order through the form controls.
  • Clear error messages. Error messages should be clear, concise, and linked to the form control they relate to.

Accessibility Features

  • Semantic HTML. The use of "form", "label", and "input" elements provides a clear, understandable structure for assistive technologies.
  • ARIA Attributes. "aria-describedby" is used to associate error messages with their corresponding form controls, providing context to screen readers.
  • Keyboard Operability. All form controls should be fully operable using a keyboard.

Testing

To ensure the form component meets accessibility standards, refer to the Form Component Testing Checklist.
