Form

This guide provides instructions for creating a form component that meets accessibility standards.

Interactive Example

Source Code

<form>

  <div class="usa-form-group">

    <label class="usa-label" for="name">Name <span class="required-asterisk">*</span></label>

    <input class="usa-input" id="name" name="name" type="text" required aria-required="true">

  </div>

  <div class="usa-form-group">

    <label class="usa-label" for="email">Email <span class="required-asterisk">*</span></label>

    <input class="usa-input" id="email" name="email" type="email" required aria-required="true">

  </div>

  <button class="usa-button" type="submit">Submit</button>

</form>

Guidance

  • Display form controls in the same order in the HTML as they appear on screen.
  • Group thematically related controls in a <fieldset> and use <legend> elements for better accessibility.
  • Use simple vertical layouts to accommodate limited vision.
  • Show required fields with a red asterisk (*) and mark optional fields as "optional".

Accessibility Features

  • Forms are keyboard-accessible and display a clear focus state.
  • Color contrast meets WCAG 2.1 AA standards for visibility.
  • Use aria-required to indicate mandatory fields for screen readers.

Testing

To ensure the form component meets accessibility standards, refer to the Form Component Testing Checklist.
