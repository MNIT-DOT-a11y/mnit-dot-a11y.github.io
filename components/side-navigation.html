Side Navigation

A side navigation component allows users to easily access different sections or pages of a website. It should be keyboard accessible and properly announce its state to screen readers.

Source Code

<nav aria-label="Primary side navigation">
    <ul class="side-nav">
        <li><a href="#section-1">Section 1</a></li>
        <li class="expander-group">
            <button class="expander-toggle" aria-expanded="false" aria-controls="sub-nav-1">Section 2
                <i class="fa-solid fa-angle-down icon" role="img" aria-label="Expand" aria-hidden="true"></i>
            </button>
            <ul id="sub-nav-1" class="sub-nav">
                <li><a href="#section-2-1">Section 2.1</a></li>
                <li><a href="#section-2-2">Section 2.2</a></li>
            </ul>
        </li>
        <li class="active"><a href="#section-3" aria-current="page">Section 3</a></li>
        <li><a href="#section-4">Section 4</a></li>
    </ul>
</nav>

Accessibility Guidance

  • Use `role="navigation"` or the `
    ` element to identify the navigation landmark.
  • Use `aria-label` on the `
    ` element for a concise label, if a visible heading is not available.
  • For collapsible menus, use a `
  • Use `aria-current="page"` on the link for the current page to inform screen readers.
  • Ensure all links are keyboard focusable and provide clear focus styles.
