Side Navigation Component

A side navigation, or vertical navigation, is a crucial component for helping users navigate within a specific section of a website. It often presents content in a hierarchical, tree-like structure, allowing users to drill down into sub-sections of a page or site.

Passed WCAG 2.1 AA

Interactive Example

Standard Side Navigation

  • Main Topic 1
  • Main Topic 2
    • Sub-item 2.1
    • Sub-item 2.2
    • Sub-item 2.3
  • Main Topic 3

Source Code

<nav aria-label="On-page navigation">
    <ul>
        <li><a href="#">Main Topic 1</a></li>
        <li><a href="#" class="active">Main Topic 2</a></li>
        <li class="sub-menu-item">
            <ul>
                <li><a href="#">Sub-item 2.1</a></li>
                <li><a href="#">Sub-item 2.2</a></li>
                <li><a href="#">Sub-item 2.3</a></li>
            </ul>
        </li>
        <li><a href="#">Main Topic 3</a></li>
    </ul>
</nav>

Guidance

Side navigation is best used for content that has a clear hierarchy or for navigating within a specific section of a large site. It should be easy to access and understand from any page.

When to use a side navigation component

  • Nested content. Ideal for documentation, help centers, or e-commerce sites where content is broken down into sub-categories.
  • Filtering. Can be used to display filter options in a search results page.

Usability tips

  • Clear labels. All navigation links should have clear and descriptive text.
  • Visual hierarchy. Use visual cues like indentation or different font weights to communicate the nested structure of the navigation.

Accessibility Features

  • Nav Landmark. The "nav" element with "aria-label" provides a clear landmark role for the navigation block.
  • Semantic Lists. Using nested "ul" elements is the correct semantic way to mark up a hierarchical navigation structure.
  • Keyboard Operability. Users should be able to navigate through all navigation items using the Tab key, and interact with expand/collapse controls using the Enter or Space key.

Testing

To ensure the side navigation component meets accessibility standards, refer to the Side Navigation Component Testing Checklist.
