Card Component

A card is a flexible and extensible content container. They are ideal for grouping related information, such as a heading and a short description, to create a clear and organized visual display.

Passed WCAG 2.1 AA

Interactive Example

Standard Card

Card Title

This is a brief description of the card's content. It provides a summary of the information the user will find if they click the link below.

Learn More

Source Code

<div class="card">
    <h4 class="card-title">Card Title</h4>
    <p class="card-text">This is a brief description of the card's content. It provides a summary of the information the user will find if they click the link below.</p>
    <a href="#" class="card-link">Learn More</a>
</div>

Guidance

Use card components when you need to display a collection of related content in a visually scannable format. This helps users quickly grasp the purpose of each item and choose the one they are interested in.

When to use the card component

  • Grouped content. Ideal for displaying a list of products, articles, or services.
  • Link previews. Use cards to provide a preview of content that links to a new page.

Usability tips

  • Clear hierarchy. Use headings and clear text to establish a strong visual hierarchy within the card.
  • Minimalism. Cards work best when they contain a concise amount of information. Avoid cluttering them with too much text or too many interactive elements.

Accessibility Features

  • **Semantic HTML.** Use proper semantic HTML, such as `h1-h6` for titles and `p` for paragraphs, to help assistive technologies understand the structure.
  • **Keyboard Operability.** If the entire card is a link, ensure it is focusable as a single unit for keyboard users.
  • **Contrast.** All text and background colors should meet WCAG 2.1 AA contrast standards. The chosen blue background and white text ensure this.

Testing

To ensure the card component meets accessibility standards, refer to the Card Component Testing Checklist.
