Pagination Component

Pagination allows users to navigate through a large set of content, such as search results or articles, by dividing it into a series of smaller, more manageable pages. This component helps to improve performance and user experience by preventing long, endless scrolling.

Passed WCAG 2.1 AA

  • 1

Source Code

<nav aria-label="Pagination">
    <ul class="pagination-list">
        <li class="pagination-item disabled"><a href="#" class="pagination-link" aria-label="Previous page">« Previous</a></li>
        <li class="pagination-item"><a href="#" class="pagination-link">1</a></li>
        <li class="pagination-item active"><a href="#" class="pagination-link" aria-current="page">2</a></li>
        <li class="pagination-item"><a href="#" class="pagination-link">3</a></li>
        <li class="pagination-item"><a href="#" class="pagination-link" aria-label="Next page">Next »</a></li>
    </ul>
</nav>

Guidance

Pagination should be used on any page with long lists of content to prevent overwhelming the user. It provides a clear and consistent way to navigate through a data set one section at a time.

When to use the pagination component

  • Search results. Ideal for displaying search results or data tables that are too large to fit on a single page.
  • Article archives. For navigating through a collection of blog posts or news articles.

Usability tips

  • Clear navigation. Provide clear "Previous" and "Next" links to help users move between pages.
  • Indicate current page. The current page should be visually and programmatically highlighted to help users orient themselves.

Accessibility Features

  • Nav Landmark. The "nav" element with an "aria-label" provides a clear landmark role for the entire pagination component.
  • Current Page. The "aria-current="page"" attribute on the current page link informs screen readers that it represents the active page.
  • Descriptive Links. The "Previous" and "Next" links have "aria-label" attributes to provide clear, descriptive text for screen readers.
  • Disabled State. The "disabled" class on the list item for the "Previous" link visually communicates that the link is not active.

Testing

To ensure the pagination component meets accessibility standards, refer to the Pagination Component Testing Checklist.
