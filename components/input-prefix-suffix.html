Input Prefix/Suffix Component

An input prefix or suffix is a non-editable string of text that appears before or after an input field, providing context for the expected value. This component is ideal for form fields that require a specific unit or currency symbol, such as a dollar sign for a price input.

Passed WCAG 2.1 AA

Interactive Example

Prefix and Suffix Inputs

$ .00

Enter the price in dollars. The input will automatically be formatted with a dollar sign prefix and a cents suffix.

kg

Enter the weight in kilograms.

Source Code

<!-- Example with both prefix and suffix -->
<div class="input-group">
    <label for="price-input" class="sr-only">Price in dollars</label>
    <span class="input-group-prefix">$</span>
    <input type="text" id="price-input" class="input-group-field" aria-describedby="price-description">
    <span class="input-group-suffix">.00</span>
    <p id="price-description" class="sr-only">Enter the price in dollars. The input will automatically be formatted with a dollar sign prefix and a cents suffix.</p>
</div>

<!-- Example with just a suffix -->
<div class="input-group">
    <label for="weight-input">Weight (kg)</label>
    <input type="text" id="weight-input" class="input-group-field" aria-describedby="weight-description">
    <span class="input-group-suffix">kg</span>
    <p id="weight-description" class="sr-only">Enter the weight in kilograms.</p>
</div>

Guidance

Input prefixes and suffixes can reduce user error by providing a visual reminder of the expected input format. They are purely presentational and should not be included in the input value itself. Proper labeling is crucial for screen readers to understand the full context of the input field.

When to use the input prefix/suffix component

  • Currency and units. Ideal for fields where the input value is always a specific unit, like currency ($) or measurements (kg, cm, etc.).
  • Standard formats. Use for fields like URL inputs where the "https://" prefix is a constant.

Usability tips

  • Read-only. Ensure the prefix or suffix is not editable by the user.
  • Associated label. The entire input group should have a clear and descriptive label associated with it.

Accessibility Features

  • Descriptive Labels. Use a visible "label" element to describe the purpose of the entire input, including the prefix/suffix. For inputs that have a label that isn't visually part of the input, the "aria-describedby" attribute links the label to the input.
  • Correct Markup. Ensure the prefix and suffix are contained within "span" or other non-interactive elements and are not focusable.

Testing
