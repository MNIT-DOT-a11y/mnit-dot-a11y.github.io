Data Visualizations Component

Data visualizations, such as charts and graphs, are powerful tools for presenting complex information in an easily digestible format. Ensuring they are accessible means providing alternative ways to access the data for users who cannot perceive the visual representation.

Passed WCAG 2.1 AA

Data table for monthly sales.

Source Code

<div class="chart-container" role="img" aria-labelledby="chart-title chart-description">
    <h4 id="chart-title">Monthly Sales Data</h4>
    <p id="chart-description">A bar chart showing sales for the first quarter of the year.</p>
    <!-- The SVG content for the chart would go here -->
    <p class="sr-only">Sales Data Table:</p>
    <table class="chart-table">
        <caption class="sr-only">Data table for monthly sales.</caption>
        <thead>
            <tr>
                <th scope="col">Month</th>
                <th scope="col">Sales (in thousands)</th>
            </tr>
        </thead>
        <tbody>
            <tr>
                <td>January</td>
                <td>200</td>
            </tr>
            <tr>
                <td>February</td>
                <td>300</td>
            </tr>
            <tr>
                <td>March</td>
                <td>250</td>
            </tr>
        </tbody>
    </table>
</div>

Guidance

The key to accessible data visualizations is providing a text alternative that conveys the same information as the visual representation. This can be a data table, a detailed description, or a combination of both.

When to use accessible data visualizations

  • Any time data is presented visually. Charts, graphs, maps, and infographics all require a text-based alternative.
  • Complex data sets. For charts with many data points, a detailed description is crucial.

Usability tips

  • Provide a table. A data table is the most robust and accessible way to provide the underlying data.
  • Summarize the key takeaway. The description should not only describe the chart but also summarize its main point.

Accessibility Features

  • Descriptive Title and Description. The "aria-labelledby" attribute links the chart to a visible title and a more detailed description, providing context for screen reader users.
  • Data Table. An accessible data table is provided as a text-based equivalent of the chart. The "sr-only" class ensures it is hidden from sighted users but available to screen readers.
  • ARIA Roles. The "role="img" on the container helps assistive technologies understand that the contained content is a single image or a chart.
