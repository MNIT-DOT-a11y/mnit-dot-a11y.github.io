Data Visualizations Component

This guide provides instructions for creating accessible data visualizations. Visualizations should not rely on color alone to convey meaning and should have alternative text or a table to make the data available to all users.

Source Code

<div class="chart-container">
  <div class="bar-chart" aria-label="Bar chart showing fictional data">
    <div class="bar" style="height: 50%;" role="img" aria-label="Category A, Value 50">
      <span class="bar-label">A</span>
    </div>
    <div class="bar" style="height: 75%;" role="img" aria-label="Category B, Value 75">
      <span class="bar-label">B</span>
    </div>
    <div class="bar" style="height: 30%;" role="img" aria-label="Category C, Value 30">
      <span class="bar-label">C</span>
    </div>
    <div class="bar" style="height: 90%;" role="img" aria-label="Category D, Value 90">
      <span class="bar-label">D</span>
    </div>
  </div>
</div>

Accessibility Guidance

  • Use `aria-label` on the chart container and individual bars to provide a description for screen readers.
  • Ensure high contrast between the bars and the chart background.
  • Provide a text-based alternative (like a data table) for complex visualizations.

Testing

To ensure the data visualizations component meets accessibility standards, refer to the Data Visualizations Component Testing Checklist.
