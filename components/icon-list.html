Icon List Component

An icon list is a series of list items where each item is preceded by a visual icon. This component is used to enhance readability and make key information or features stand out. Icons should be used thoughtfully and consistently to avoid visual clutter and confusion.

Passed WCAG 2.1 AA

Interactive Example

Standard Icon List

  • Feature 1: A great feature of this product.
  • Feature 2: Another excellent feature.
  • Feature 3: The final key feature.

Source Code

<ul class="icon-list">
    <li class="icon-list-item">
        <i class="fa-solid fa-check" aria-hidden="true"></i>
        Feature 1: A great feature of this product.
    </li>
    <li class="icon-list-item">
        <i class="fa-solid fa-check" aria-hidden="true"></i>
        Feature 2: Another excellent feature.
    </li>
    <li class="icon-list-item">
        <i class="fa-solid fa-check" aria-hidden="true"></i>
        Feature 3: The final key feature.
    </li>
</ul>

Guidance

Use an icon list to present information in a way that is easy to scan and understand. Ensure that the icons are either purely decorative or that their meaning is also communicated through the accompanying text.

When to use the icon list component

  • Feature lists. For highlighting key features or benefits of a product or service.
  • Step-by-step instructions. To visually represent each step in a process.

Usability tips

  • Keep it simple. Use a limited number of icons and ensure they are easily recognizable.
  • Avoid icon-only. Never rely on icons alone to convey meaning. Always include descriptive text.

Accessibility Features

  • Semantic List. Using "ul" and "li" elements ensures that the content is announced as a list by screen readers, providing important context.
  • Decorative Icons. The "aria-hidden="true"" attribute on the "i" element ensures that screen readers ignore the decorative icon, preventing redundancy or confusion with the list item's text.
  • Keyboard Operability. The links or other interactive elements within the list items should be fully keyboard-operable.

Testing

To ensure the icon list component meets accessibility standards, refer to the Icon List Component Testing Checklist.
