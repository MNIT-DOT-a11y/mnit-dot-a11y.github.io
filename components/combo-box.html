Combo Box Component

A combo box is a text input field with an associated dropdown list that helps users select a value from a predefined list or type in their own. This component is ideal for scenarios where users need both the flexibility of a text field and the convenience of a list of common options.

Passed WCAG 2.1 AA

Interactive Example

Standard Combo Box

  • Apple
  • Banana
  • Orange
  • Grape

Source Code

<div class="combobox-wrapper">
    <label for="fruit-input">Select a fruit or type your own:</label>
    <input type="text" id="fruit-input" role="combobox" aria-autocomplete="list" aria-controls="fruit-list" aria-expanded="false">
    <ul id="fruit-list" class="combobox-list" role="listbox" hidden>
        <li class="combobox-option" role="option" id="fruit-option-1">Apple</li>
        <li class="combobox-option" role="option" id="fruit-option-2">Banana</li>
        <li class="combobox-option" role="option" id="fruit-option-3">Orange</li>
        <li class="combobox-option" role="option" id="fruit-option-4">Grape</li>
    </ul>
</div>

Guidance

Use a combo box when you want to provide users with a list of suggestions as they type. This helps speed up data entry and reduces potential errors, while still allowing for custom input.

When to use the combo box component

  • Search with suggestions. Ideal for search fields that offer a list of popular or recent search terms.
  • Filtering. Use for filtering a large data set by a specific, predefined category.

Usability tips

  • Visible feedback. The dropdown list should be clearly visible and positioned correctly when the user interacts with the input field.
  • Keyboard support. Users should be able to navigate the list of options with arrow keys and select an option with the Enter key.

Accessibility Features

  • ARIA Attributes. The `role="combobox"`, `aria-autocomplete="list"`, `aria-controls`, and `aria-expanded` attributes are essential for screen readers to understand the component's purpose and state.
  • Descriptive Labeling. The `label` provides a clear name for the input field.
  • Live Updates. When the user types, dynamic updates to the dropdown list should be announced to screen readers.

Testing

To ensure the combo box component meets accessibility standards, refer to the Combo Box Component Testing Checklist.
