Accessible Text Input

This guide provides instructions for creating a text input component that meets accessibility standards.

Interactive Example

Source Code

<label class="usa-label" for="textInput">Text input label</label>
<input class="usa-input" id="textInput" name="textInput" type="text" />

<label class="usa-label" for="textArea">Text area label</label>
<textarea class="usa-textarea" id="textArea" name="textArea"></textarea>

Guidance

  • Use text inputs for unpredictable or freeform responses where user input can vary significantly.
  • Avoid placeholder text as it can disappear when a user starts typing, which may be problematic for some users.
  • Ensure text input fields are appropriately sized to match the expected input length.
  • Consider mobile users and test the usability of text inputs on smaller screens.

Accessibility Features

  • Text inputs are designed to be keyboard-accessible and include clear focus indicators.
  • All form controls are appropriately labeled for screen readers with label elements.
  • Color contrast meets WCAG 2.1 AA standards for all input fields and labels.
