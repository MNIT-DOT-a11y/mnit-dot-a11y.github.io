Header Component

The header is the top section of a website that serves as a global navigation and branding element. An accessible header provides a clear and consistent way for users to navigate your site and understand its purpose from any page.

Guidance

The header is a critical landmark on every webpage. A well-designed header provides a clear and consistent way for users to find the site's identity, navigate, and access core functionality.

When to use the header component

  • On every page. The header should be present and consistent on every page of your website.
  • Global navigation. It is the primary location for the site's main navigation menu.

Usability tips

  • Clear branding. The site logo or name should be the most prominent element in the header.
  • Consistent layout. The header's structure should not change from page to page.

Accessibility Features

  • Landmark Role. The "header" element has an implicit "role="banner"" landmark role, which helps screen reader users quickly navigate to the top of the page.
  • Clear Navigation. The navigation within the header uses "nav" and "aria-label="Site menu"" to provide a clear label for assistive technologies.
  • Skip Links. A skip link at the top of the page should allow users to bypass the header and go directly to the main content.

Testing

To ensure the header component meets accessibility standards, refer to the Header Component Testing Checklist.
