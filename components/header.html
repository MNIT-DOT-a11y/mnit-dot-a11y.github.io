Accessible Header

This guide provides instructions for creating a header component that meets accessibility standards.

Interactive Example

<header class="usa-header">
  <div class="usa-nav-container">
    <div class="usa-logo">
      <em class="usa-logo__text"><a href="/" title="Project title">Project Title</a></em>
    </div>
    <nav class="usa-nav" aria-label="Primary navigation">
      <ul class="usa-nav__primary">
        <li class="usa-nav__item"><a href="#section1" class="usa-nav__link">Section 1</a></li>
        <li class="usa-nav__item"><a href="#section2" class="usa-nav__link">Section 2</a></li>
        <li class="usa-nav__item"><a href="#section3" class="usa-nav__link">Section 3</a></li>
      </ul>
      <div class="usa-search">
        <input type="search" class="usa-search__input" placeholder="Search">
        <button class="usa-search__button" type="submit">Search</button>
      </div>
    </nav>
  </div>
</header>

Guidance

  • Include skip navigation links to allow users with screen readers to bypass long navigation lists.
  • Ensure that navigation is keyboard compatible, allowing users to use Tab to navigate.
  • Use list elements for navigation links to enhance screen reader navigation.
  • Add alt text to logo images if they are not text.

Accessibility Features

  • Headers are designed to be keyboard-accessible with clear focus indicators.
  • Color contrast meets WCAG 2.1 AA standards for visibility.
  • Structured menu navigation with clear labeling for screen readers.
