Guidance

Use buttons to perform an action on the current page, such as submitting a form, toggling a section, or triggering a dialog. Avoid using buttons to navigate to a new page; use a link for that purpose.

When to use the button component

  • Form submission. Buttons are the standard for submitting form data.
  • Interactive actions. Use buttons to show/hide content, open a modal dialog, or play/pause media.

Usability tips

  • Clear labels. The text within the button should clearly describe the action that will occur, e.g., "Submit", "Save", "Cancel", "Send".
  • Visible focus. Ensure buttons have a clear, visible focus state for keyboard users.

Accessibility Features

  • Semantic HTML. The `<button>` element is inherently accessible to assistive technologies.
  • Keyboard operable. Buttons are focusable and can be activated using the Enter or Space key.
