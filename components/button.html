Accessible Button

This guide provides instructions for creating a button component that meets accessibility standards.

Interactive Example

Source Code

<button class="usa-button" type="button">Default</button>

<button class="usa-button" type="button" disabled>Disabled</button>

Guidance

  • Use buttons for the most important actions like Download, Sign up, or Log out.
  • Ensure the button displays a visible focus state when users tab to them.
  • Avoid using non-semantic tags like <div> or <img> for buttons.
  • Use the type attribute to define the button's behavior.
  • Use disabled or aria-disabled to signify disabled buttons.

Accessibility Features

  • Buttons are designed to be keyboard-accessible and have clear focus indicators.
  • Color contrast meets WCAG 2.1 AA standards to ensure visibility for users with visual impairments.
  • Disabled buttons are visually distinct to indicate they cannot be interacted with.

Testing

To ensure the button component meets accessibility standards, refer to the Button Component Testing Checklist.
