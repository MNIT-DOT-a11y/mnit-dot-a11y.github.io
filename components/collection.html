Collection Component

A collection is a container used to display a group of related items. It is often used for displaying articles, products, or services in an organized way, such as a grid of cards or a list of links.

Passed WCAG 2.1 AA

Source Code

<div class="collection" role="list">
    <div class="collection-item" role="listitem">
        <a href="#" class="collection-card">
            <h4 class="collection-card-title">Item 1</h4>
            <p>This is a description of the first item in the collection.</p>
        </a>
    </div>
    <div class="collection-item" role="listitem">
        <a href="#" class="collection-card">
            <h4 class="collection-card-title">Item 2</h4>
            <p>This is a description of the second item in the collection.</p>
        </a>
    </div>
    <div class="collection-item" role="listitem">
        <a href="#" class="collection-card">
            <h4 class="collection-card-title">Item 3</h4>
            <p>This is a description of the third item in the collection.</p>
        </a>
    </div>
</div>

Guidance

Use collections to group related content in a visually scannable format. This helps users quickly grasp the purpose of each item and choose the one they are interested in.

When to use the collection component

  • Product listings. Ideal for displaying a list of products, articles, or services.
  • Gallery views. Can be used to present a gallery of images, where each image is an item in the collection.

Usability tips

  • Consistent style. All items within a collection should have a similar visual style and content structure for consistency.
  • Clear actions. If the collection items are links, ensure the entire item is clickable to improve the target size for users with mobility impairments.

Accessibility Features

  • Semantic HTML. Using "div`s with "role="list" and "role="listitem" ensures screen readers announce the collection as a list, which provides context.
  • Keyboard Operability. Ensure all interactive elements within the collection are reachable and operable via a keyboard.
  • Descriptive Links. Links or buttons within the collection should have clear and descriptive text that explains their purpose.

Testing

To ensure the collection component meets accessibility standards, refer to the Collection Component Testing Checklist.
