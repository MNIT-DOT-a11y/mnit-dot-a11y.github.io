Modal Component

A modal is a temporary window that overlays a webpage. It requires user interaction before returning to the main page content, and is useful for critical alerts or short-form tasks.

Interactive Example

Modal Title

This is the content of the modal. You must close this window to continue.

Source Code

<button class="usa-button" onclick="document.getElementById('modal-example').showModal()">Open Modal</button>

<dialog id="modal-example" class="modal" aria-labelledby="modal-heading">
  <h2 id="modal-heading">Modal Title</h2>
  <p>This is the content of the modal. You must close this window to continue.</p>
  <button class="usa-button" onclick="document.getElementById('modal-example').close()">Close</button>
</dialog>

Accessibility Guidance

  • Semantic HTML: Use the `` element for native modal functionality. It handles focus trapping, modality, and the ESC key by default, which is far more robust and accessible than building a modal from scratch with `div`s.
  • `aria-labelledby`: The `aria-labelledby` attribute on the `` element must reference the `id` of the modal's title. This provides an accessible name for the modal window.
  • Focus Management: When the modal opens, focus should be moved to the first interactive element inside it (or the modal itself). When the modal closes, focus should be returned to the element that triggered it. The `` element handles this automatically.
  • Keyboard Interaction: Users must be able to close the modal using the **Escape** key. This is a crucial feature for keyboard and screen reader users, and it is handled natively by the `` element.
  • Background Content: The content behind the modal should be made inaccessible to assistive technologies. The `` element handles this naturally. If you are not using ``, you must use `aria-hidden="true"` on the main page content to prevent screen readers from interacting with it.
