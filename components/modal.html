Modal Component

A modal is a temporary pop-up window that appears over the main content, demanding user attention or input before they can continue. Accessible modals are designed to be fully usable for all users, including those who rely on a keyboard or screen reader.

Passed WCAG 2.1 AA

Modal Title

This is the content of the modal. You can place forms, alerts, or other information here.

Source Code

<!-- Button to trigger the modal -->
<button class="usa-button usa-button--primary" onclick="openModal()">Open Modal</button>

<!-- The modal structure -->
<div id="modal-overlay" class="modal-overlay" hidden>
    <div id="modal-dialog" class="modal-dialog" role="dialog" aria-labelledby="modal-title" aria-modal="true">
        <h4 id="modal-title">Modal Title</h4>
        <p>This is the content of the modal.</p>
        <button class="modal-close-button" onclick="closeModal()">
            <span aria-hidden="true">×</span>
            <span class="sr-only">Close modal</span>
        </button>
        <div class="modal-actions">
             <button class="usa-button usa-button--secondary" onclick="closeModal()">Cancel</button>
             <button class="usa-button usa-button--primary" onclick="closeModal()">Confirm</button>
        </div>
    </div>
</div>

Guidance

Modals are useful for capturing user attention without navigating away from the current page. They should be used sparingly for critical tasks, such as confirmations or alerts that require a direct response.

When to use a modal component

  • User confirmations. To confirm a user's action, such as deleting an account.
  • Forms or wizards. For multi-step forms or small forms that need to be completed before the user can proceed.

Usability tips

  • Keyboard trap. When the modal is open, focus should be confined to the modal itself, preventing users from accidentally tabbing to elements in the background.
  • Clear dismissals. Provide multiple ways to close the modal, including an "x" button, a "Cancel" button, and the Escape key.

Accessibility Features

  • ARIA Roles. The "role="dialog"" and "aria-modal="true"" attributes are crucial for screen readers to announce the modal as a dialog and to understand that the background content is no longer interactive.
  • Keyboard Focus. The modal should trap the keyboard focus, ensuring that users can't tab to elements outside of the modal when it's open. The Escape key should also close the modal.
  • Descriptive Labels. The modal's title should be linked to the dialog using "aria-labelledby" to provide a clear and accessible name.
