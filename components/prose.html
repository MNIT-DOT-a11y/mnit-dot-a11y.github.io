Prose Component

A prose component refers to the standard blocks of rich text content that make up the bulk of a webpage. This can include paragraphs, headings, lists, bold or italic text, and blockquotes. Ensuring this content is accessible is fundamental to making a website usable for everyone.

Passed WCAG 2.1 AA

Guidance

Using correct semantic markup is the most important step for making prose content accessible. Avoid using generic `div` and `span` tags for structure; instead, use tags like `p`, `h1`-`h6`, `ul`, `ol`, and `blockquote` to convey meaning to screen readers.

When to use prose components

  • Any written content. Prose is the foundation of most web pages, including articles, blog posts, and documentation.
  • Informational sections. Used to convey any information that isn't part of a specific interactive component.

Usability tips

  • Font readability. Choose a font with a clear and legible typeface, and use a comfortable font size and line height.
  • Color contrast. Ensure text has sufficient contrast against its background.
  • Clear hierarchy. Use headings (`h1` through `h6`) in a logical order to structure your content.

Accessibility Features

  • **Semantic Tags.** Using tags like `

    `, `

    `, and `
      ` ensures that assistive technologies can understand the content's structure and navigate it correctly.
    • **Readable Font.** The text should be resizable and readable at various sizes without losing information.
    • **Color Contrast.** The default text and background colors have been chosen to meet WCAG 2.1 AA contrast standards.
