File Input Component

A file input component allows a user to select one or more files from their local device for upload. It's a standard form element, but can be customized to provide a more consistent visual experience across browsers.

Passed WCAG 2.1 AA

Source Code

<!-- Standard File Input -->
<div class="file-input-wrapper">
    <label for="file-upload" class="file-input-label">Choose a file to upload:</label>
    <input type="file" id="file-upload" aria-describedby="file-upload-feedback">
    <div id="file-upload-feedback" class="file-input-feedback" aria-live="polite">No file chosen.</div>
</div>

<!-- Customized File Input -->
<div class="file-input-wrapper">
    <label for="custom-file-upload" class="file-input-label">Upload a document:</label>
    <input type="file" id="custom-file-upload" class="sr-only" aria-describedby="custom-file-upload-feedback">
    <button class="file-input-button" onclick="document.getElementById('custom-file-upload').click();">Choose File</button>
    <div id="custom-file-upload-feedback" class="file-input-feedback" aria-live="polite">No file chosen.</div>
</div>

Guidance

The file input component is crucial for forms that require document uploads. Proper labeling and dynamic feedback are essential for a good user experience and accessibility.

When to use the file input component

  • Form submissions. When a form requires users to upload documents, images, or other files.
  • Profile customization. For allowing users to upload a profile picture or custom avatar.

Usability tips

  • Clear feedback. The name of the selected file(s) should be displayed to the user for confirmation.
  • Instructions. Provide clear instructions on what file types or sizes are allowed.

Accessibility Features

  • **Descriptive Labels.** The `label` is correctly associated with the `input` using `for` and `id` attributes.
  • **Keyboard Operability.** The standard file input is natively keyboard-operable. For customized inputs, the button trigger must be keyboard-focusable.
  • **Live Region.** The feedback `div` uses `aria-live="polite"` to announce the selected file to screen readers without disrupting the user.

Testing

To ensure the file input component meets accessibility standards, refer to the File Input Component Testing Checklist.
