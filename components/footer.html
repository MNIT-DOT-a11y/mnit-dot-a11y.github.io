Footer Component

A footer is located at the bottom of a webpage and contains important links, copyright information, and contact details. An accessible footer helps users find key information and navigate the site efficiently, regardless of how they access the content.

Passed WCAG 2.1 AA

Source Code

<footer class="container" id="footer" role="contentinfo">
    <nav aria-label="Footer navigation">
        <ul>
            <li><a href="/about/">About</a></li>
            <li><a href="/contact/">Contact Us</a></li>
            <li><a href="/privacy/">Privacy Policy</a></li>
            <li><a href="/site-map/">Site Map</a></li>
        </ul>
    </nav>
    <p>MNIT DOT Accessibility Testing Resources. Source code by the T-Mobile - Accessibility Resource Center</p>
</footer>

Guidance

The footer is a critical part of the user experience, providing key information and navigational links that are available on every page. Ensure your footer is consistently designed and easy to navigate.

When to use the footer component

  • On every page. The footer should be present on every page of your website to provide consistent access to important information.
  • Legal and contact info. It is the standard location for legal notices, copyright, and contact details.

Usability tips

  • Semantic structure. Use the "footer" tag to clearly define the footer region.
  • Organized navigation. Use "aria-label" on any navigation within the footer to provide context for screen readers.

Accessibility Features

  • Landmark Role. The "footer" element has an implicit "role="contentinfo"" landmark role, which helps screen reader users quickly navigate to the end of the page.
  • Logical Structure. The use of "nav" and "ul" elements ensures the navigation links are presented in a clear, list-based format.
  • Skip Links. A skip link to the footer helps users bypass the main content and jump directly to the footer for easy access.

Testing

To ensure the footer component meets accessibility standards, refer to the Footer Component Testing Checklist.
