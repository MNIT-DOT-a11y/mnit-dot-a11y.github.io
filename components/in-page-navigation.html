Accessible In-Page Navigation

This guide provides instructions for creating an in-page navigation component that meets accessibility standards.

Source Code

<div class="usa-in-page-nav-container">

  <aside class="usa-in-page-nav" aria-labelledby="nav-title">

    <h4 id="nav-title">On this page</h4>

    <a href="#section1">Section 1</a>

    <a href="#section2">Section 2</a>

    <a href="#section3">Section 3</a>

  </aside>

  <main id="main-content" class="main-content usa-prose">

    <h2 id="section1">Section 1</h2>

    <p>Content for section 1...</p>

    <h2 id="section2">Section 2</h2>

    <p>Content for section 2...</p>

    <h2 id="section3">Section 3</h2>

    <p>Content for section 3...</p>

  </main>

</div>

Guidance

  • Use in-page navigation for long pages with multiple sections to improve the user experience.
  • Ensure that the navigation links reflect the corresponding section headings on the page.
  • Place the in-page navigation in a sticky position beside the main content for easy access.
  • Ensure the in-page navigation is keyboard-accessible, allowing users to navigate using the `Tab` key.

Accessibility Features

  • Focus states are clearly defined for keyboard navigation.
  • Link text is descriptive and matches the corresponding section headings.
  • Color contrast in links meets WCAG 2.1 AA standards for visibility.

Testing

To ensure the in-page navigation component meets accessibility standards, refer to the In-Page Navigation Component Testing Checklist.
