In-page Navigation Component

In-page navigation, often in the form of a table of contents or "jump links," helps users quickly navigate to specific sections of a long document or webpage. For accessibility, it is crucial that these links are properly structured and provide a clear way to jump to the intended content.

Passed WCAG 2.1 AA

Standard In-page Navigation

  • Section 1: The First Topic
  • Section 2: The Second Topic
  • Section 3: The Third Topic

Section 1: The First Topic

This is the content for the first section. In-page navigation links are essential for long pages, as they allow users to quickly find the information they are looking for without scrolling. This improves both usability and accessibility. Proper use of semantic headings ("h2", "h3", etc.) is key to creating anchor points for these links.

Section 2: The Second Topic

This is the content for the second section. Screen readers can use the list of navigation links to jump directly to the content, which saves time and effort. For sighted users, the navigation provides a quick overview of the page's structure.

Section 3: The Third Topic

This is the content for the third section. Ensure that the text of the in-page navigation link clearly matches the heading of the section it links to. This consistency is vital for all users.

Source Code

<!-- In-page navigation menu -->
<nav aria-label="On-page links">
    <ul>
        <li><a href="#section1">Section 1: The First Topic</a></li>
        <li><a href="#section2">Section 2: The Second Topic</a></li>
        <li><a href="#section3">Section 3: The Third Topic</a></li>
    </ul>
</nav>

<!-- Content sections with matching IDs -->
<div id="section1">
    <h3>Section 1: The First Topic</h3>
    <p>...</p>
</div>

<div id="section2">
    <h3>Section 2: The Second Topic</h3>
    <p>...</p>
</div>

<div id="section3">
    <h3>Section 3: The Third Topic</h3>
    <p>...</p>
</div>

Guidance

In-page navigation links are essential for providing a better user experience on long, content-heavy pages. They serve as a quick reference, allowing users to jump directly to the information they need.

When to use the in-page navigation component

  • Long documents. Use on any page with a lot of content that is broken up into distinct sections.
  • Tables of contents. Ideal for creating a table of contents that allows users to quickly scan and access information.

Usability tips

  • Match headings. The text of the navigation link should accurately reflect the heading of the section it links to.
  • Visual feedback. Consider highlighting the link for the section the user is currently viewing to provide a clear sense of location.

Accessibility Features

  • Nav Landmark. The "nav" element with "aria-label" correctly identifies the component as a landmark region for screen readers.
  • Descriptive Links. The "href" attribute on the links points to the "id" of the destination heading, which is a standard and accessible method of in-page navigation.
  • Visual Focus. All links must have a visible focus state for keyboard users.

Testing

To ensure the in-page navigation component meets accessibility standards, refer to the In-page Navigation Component Testing Checklist.
