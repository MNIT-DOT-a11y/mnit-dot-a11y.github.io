1 Determine the Scope for Testing

Work with the Project Team to complete the accessibility scoping template to determine what will be tested.

2 Plan for Testing

Prioritizes critical user journeys, core functionality, and high-risk areas.

Note: Plan on regularly reviewing and updating the testing plan to accommodate changes and insights as you progress through the testing process.

3 Conduct Testing

Automated Tests: Use Axe DevTools to find issues that can be found automatically. Record the findings in the report in the Accessibility Test Report spreadsheet.

    • Test the identify application-wide elements and critical workflows.
    • Identify common accessibility issues such as missing alt text, contrast problems, and other programmatically detectable defects.

Manual Tests: Manual testing finds issues that automated testing can not. Record the findings in the report in the Accessibility Test Report spreadsheet.

  • Test all functionality using one keyboard input.
  • Ensure proper focus management and logical tab order.
  • Verify there are no keyboard traps.
  • Color Contrast
    • Visual inspect proper color contrast using the CCA tool.
  • Verify text resizing and reflow.
  • Access the clarity of form labels and instructions.
  • Browser Zoom Test
    • Zoom the browser to 200%
    • Review the screen being tested, looking for elements not initially displayed (e.g., collapsed sections, pop-up dialogs), display as many as possible
  • Visual Inspection at 200% Zoom
    • Check for text that did not increase in size by 200%.
    • Verify that text does not get cut off, overlap, or become unreadable
    • Ensure all content is still present and in a logical order.
    • Check if page navigation adapts appropriately (e.g., mobile-style menus)
    • Verify that the layout adapts to the new viewing dimensions.
    • Check if reflow caused horizontal scrolling.
  • Functionality at 200% Zoom
    • Verify that all interactive elements (links, buttons, forms, menus) still function correctly.
    • Ensure that visible focus indication remains for all active elements.
    • Test keyboard navigation to confirm it still works as expected.
  • High Contrast Mode
    • Activate Windows High Contrast theme (left Alt + left Shift + PrtSc)
    • Verify that text takes on high-contrast colors
    • Check that information conveyed by color is still distinguishable.
  • Magnification
    • Test with screen magnification software set to 400%.
    • Confirm that screen readers can still navigate and read content correctly.
    • Verify functionality that all interactive elements (buttons, links, forms) remain functional when scaled.
  • Verify the alt text provided for the images is appropriate alt text.
  • Ensure content remains readable and accessible with adjusted spacing.
  • Heading Structure
    • Verify logical and hierarchical heading structure (H1-H6)
    • Test navigation of headings using screen readers
  • Landmark Regions
    • Ensure proper use of HTML5 semantic elements or ARIA landmarks.
    • Test navigation between landmark regions using screen readers.
  • Reading Order
    • Verify that the reading order in the DOM matches the visual layout.
    • Test the logical flow of information for screen reader users.
  • Functionality
    • Verify all form controls are operable with keyboard and assistive technologies.
    • Test error handling and validation messages.
    • Ensure form labels are appropriately associated with inputs.
  • Dynamic Behavior
    • Test any dynamically appearing forms fields or sections.
    • Verify that screen readers announce error messages.
  • ARIA Implementation
    • Verify the correct use of ARIA roles, states, and properties.
    • Test custom widgets with various assistive technologies.
  • Keyboard Interaction
    • Ensure custom widgets follow expected keyboard interaction patterns.
    • Test for proper focus management within complex widgets.
    • Verify there are no keyboard traps.
  • Captions and Transcripts
    • Verify the presence and accuracy of captions for video content.
    • Ensure audio-only content has accurate transcripts.
  • Media Player Accessibility
    • Test media player controls for keyboard and screen reader accessibility.
    • Verify the presence of audio descriptions for visual information in videos.
  • Instant Page Updates (maps, PowerBI charts).
    • Test dynamic content updates for proper screen reader announcements.
    • Verify that focus is managed correctly after content changes.
  • Infinite Scrolling
    • Ensure keyboard users can access all dynamically loaded content.
    • Test screen reader announcement of newly loaded items.

Assistive Technology Testing: Testing with Assistive Technology. Record the findings in the report in the Accessibility Test Report spreadsheet.

  • Test with Chrome or Edge and Firefox.
  • Verify proper heading structure, landmark regions, and ARIA attributes.
  • Test with Windows Magnification
  • Ensure content remains readable and functional when magnified.

4 Complete Accessibility Report

Review and reported accessibility findings using the Accessibility Test Report Spreadsheet, ensuring each issue is accurately identified and documented, along with recommended remediations. After review, package the reports and supporting materials to share with project teams and stakeholders, facilitating accurate communication.

The package should include:

  • Accessibility Scoping document.
  • Accessibility Test Report Spreadsheet.
  • Supporting materials such as screenshots, code snippets, reference guidelines.
  • Test reports from Axe DevTools and Acrobat PDF checker.
  • Executive Summary (if requested)

5 Plan for Remediation

Discuss the testing results with the project team and prioritize remediation issues based on risk, severity, business value, etc.

6 Conduct Regression Testing

Re-run previously run tests to check remediation and identify new issues that may arise.
