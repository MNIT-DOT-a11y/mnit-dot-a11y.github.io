Power BI relies primarily on standard web keyboard navigation, with a few app-specific behaviors. Below is a reference for keyboard testing published reports and dashboards.

Complete list of keyboard shortcuts in Power BI Desktop.

General tips for accessibility testing

When keyboard testing:

  • Navigate the entire experience with keyboard only, ensuring all interactive elements are reachable, operable, and have visible focus.
  • Check that all interactive elements (filters, visuals, buttons) are reachable and usable without a mouse.
  • Make sure users can exit any component (like a visual or filter pane) without getting stuck.
  • While in Focus Mode, verify data tables are well-structured with a table header and rows identified.
