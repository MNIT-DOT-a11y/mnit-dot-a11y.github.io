Power BI relies primarily on standard web keyboard navigation, with a few app-specific behaviors. Below is a reference for keyboard testing published reports and dashboards.

Complete list of keyboard shortcuts in Power BI Desktop.

General tips for accessibility testing

When keyboard testing:

  • Navigate the entire experience with keyboard only, ensuring all interactive elements are reachable, operable, and have visible focus.
  • Check that all interactive elements (filters, visuals, buttons) are reachable and usable without a mouse.
  • Make sure users can exit any component (like a visual or filter pane) without getting stuck.
  • While in Focus Mode, verify data tables are well-structured with a table header and rows identified.

Core Navigation Keys

Shortcut Action
Shift + / Show all keyboard shortcuts
Ctrl + F6 Move focus between major report sections (like slicers, visuals).
Tab Move forward through focusable elements within a section or visual.
Shift + Tab Move backward through elements.
Enter Enter visuals (Enter), move within them
Esc Exit the current visual/section and return focus to the main report area.

Visual Interaction & Data Selection

Shortcut Action
Enter / Spacebar Select or activate a data point or element.
Ctrl + Spacebar Multi-select data points.
Esc Exit a visual or layer.
Alt + Shift + F10 Move focus to the visual menu.
Alt + Shift + F11 Show data as a table (Focus Mode)
Enter or Spacebar;
Multi-select: Ctrl + Spacebar		 Select or clear the selection of a data point
Ctrl + H Show data point details (useful for screen readers).
Ctrl + Shift + C Clear all filter selections.

Table Navigation

Shortcut Action
Arrow keys Move focus up/down/left/right.
Ctrl + Left Arrow
Ctrl + Right Arrow		 Jump to first/last cell in row.
Ctrl + Up Arrow / Ctrl + Down Arrow Jump to first/last cell in column.
