It's important to understand that individuals with low vision have various methods to enlarge content. They may use the zoom feature in a browser to make text larger or a screen magnifier like Windows Magnifier to adjust the default font size, modify text spacing on a website, or increase the entire screen. Each method behaves differently, so it's crucial to ensure that important content remains accessible no matter which method is used.

Three different WCAG Success Criteria—resize text, reflow, and text spacing—collectively ensure that enlarging content does not compromise accessibility, maintaining usability for individuals with visual impairments.

These tests should be performed using Chrome, Edge, or Firefox browsers.

Browser Zoom vs. Windows Magnifier

Browser Zoom:

  • Purpose: Simulates how low-vision or mild visual impairments users might adjust the zoom level within a browser to read content more easily. The tests allow the tester to "see" how scaling fonts impacts readability and content flow as a browser zoom scales the entire page.
  • Use Cases:
    • Responsive Design: Ensures the layout adapts correctly at different zoom levels.
    • Text Readability: Checks if the text remains legible and doesn't overlap or get cut off.
    • Interactive Elements: Verifies that buttons, links, and forms are accessible and functional when zoomed in.

Windows Magnifier:

  • Purpose: Provides a system-wide magnification, useful for users who need to zoom into specific screen parts. The test simulates the experience for someone severely visually impaired by using extreme magnification, often only showing a small part of the screen at a time.
  • Use Cases:
    • Accessibility Testing: Ensures that all application parts are accessible when magnified.
    • Pixel-Level Inspection: Useful for checking the clarity of images and icons at high magnification levels.
    • Navigation: Tests how easily users can navigate the application using the magnifier.
    • Cursor magnification: Evaluate webpages for issues that may only arise when magnified to 400% zoom or higher.

Key Differences:

  • Scope: Browser zoom affects only the content within the browser, while Windows Magnifier affects the entire screen. This makes Windows Magnifier the sole tool for evaluating the low-vision accessibility of desktop applications where Browser Zooming is unavailable.
  • User Experience: Browser Zoom is more about adjusting the view within a single application, whereas Windows Magnifier enhances visibility across all applications.
  • Differing visual impairments: Browser Zoom is typically used by those with less severe visual impairments. At the same time, Windows Magnifier magnifies certain parts of the screen, commonly required by those with more severe vision loss.

Testing with both tools ensures comprehensive accessibility and usability for users with varying needs.

Windows Magnifier

How to test using Windows Magnifier.

  1. Launch Windows Magnifier
    • Press the Windows logo key + Plus sign (+) to open Magnifier.
    • Alternatively, open Magnifier from Settings > Ease of Access > Magnifier
  2. Navigate to the page your testing.
    • Open the page you are testing
    • Verify you view the page using Full-screen mode.
    • Increase the zoom level to a minimal 400% by using the Plus sign (+).
    • Use the mouse or keyboard to navigate through the content.
  3. Check Readability and Layout
    • Text Readability: Ensure all text, including headings and body text, is readable and images are clear at different zoom levels.
    • Layout integrity: Verify that the layout does not break or overlap when magnified.
  4. Test Interactive Elements
    • Forms and Buttons: Ensure that all interactive elements like buttons, forms, and links are functional. Mouse hover does not trigger an event, and tooltips do not block important information and can be dismissed.
    • Navigation:Use arrow keys to move around the screen.
      • Press Crtl + Alt + arrow keys to move by screen edges.
      • Ensure the navigation menu fully functions at higher magnification levels.
  5. Simulate User Scenarios
    • Perform common tasks such as reading an article, filling out a form, or navigating a menu. Note any issues with usability or accessibility that arise at various zoom levels.
      • Advanced Testing Scenarios
        • Color Inversion: Press Ctrl + Alt + I to invert colors. Verify that your webpage is still usable with inverted colors.
        • Smooth Edges: In Magnifier settings, toggle Smooth edges of images and text. Test readability in both modes.
        • Focus Tracking: Ensure the Magnifier follows your keyboard focus and text cursor as you navigate the page.
        • Zoom Increments: Test with different zoom increments (Settings > Ease of Access > Magnifier > Change zoom increments). Verify smooth transitions between zoom levels.
  6. Document Findings
    • Record any issues you encounter, including descriptions and screenshots if possible. Highlight problems with readability, layout, or navigation that must be addressed.
      • Possible issues to look for include:
        • Text becoming unreadable at certain zoom levels.
        • Images are pixelating or losing clarity when magnified.
        • Interactive elements were becoming difficult to use or access.
        • Inconsistent behavior across different Magnifier views.
  7. Provide Feedback and Suggestions
    • Share your findings and suggestions for improvement with the development team. Ensure that the solution aligns with accessibility best practices.

Browser Zoom

How to test

  1. Launch either Chrome or Firefox
  2. Navigate the Page
  3. Adjust Zoom Levels
    • Start at 100% zoom (default)
    • Gradually increase to 125%m 150%, 200% up to 400%.
      • Keyboard Shortcuts:
        • Zoom in: Press Ctrl + Plus sign (+).
        • Zoom out: Press Ctril + Minus sign(-)
        • Reset Zoom: Press Ctrl + 0
      • Browser Menu:
        • Chrome/Firefox/Edge: Click the three-dots menu (top right corner), then use the zoom options.
    • Reviewing the page at each increase. Check that content remains readable without horizontal scrolling.
  4. Verify that all text remains clear, sharp, and legible.
    • Ensure that headings, body text, and labels scale appropriately without distortion.
  5. Verify Layout Integrity:
    • Ensure the layout remains consistent and logical.
    • Check for any overlapping elements or content that gets cut off.
  6. Test Interactive Elements:
    • Forms and Buttons: Ensure all interactive elements are accessible and properly labeled.
    • Navigation: Verify that menus, links, and buttons are fully functional and navigable at various zoom levels.
  7. Simulate User Scenarios:
    • Perform typical user tasks (e.g., reading an article or filling out a form).
    • Note any accessibility barriers or usability issues at different zoom levels.
  8. Document Findings:
    • Record any issues discovered during testing, including screenshots if possible.
    • Describe problems with readability, layout, or navigation and suggest potential fixes.
      • Possible issues to look for include:
        • All content visible at 100% zoom is still available, readable, and without horizontal scrolling.
        • No content overlaps or becomes obscured.
        • Text increases in size appropriately.
        • All functionality remains usable.
        • If new tools appear (e.g., hamburger menu for mobile users), ensure they work accessibly.
  9. Provide Feedback:
    • Share your findings and recommendations with the development team to ensure necessary adjustments are made.

Text spacing

Text spacing can be tested using the text spacing bookmarklet in any browser. This bookmarklet helps users and testers verify or achieve text spacing on a website. It is recommended that you bookmark the bookmarklet for quick access.Text spacing testing includes line height, letter, word, and paragraph spacing..

How to test

  1. Set Up the Text Spacing Bookmarklet:
    • Install the Test Spacing Bookmarklet
    • Drag the Text Spacing Bookmarklet to your browser’s bookmark bar. Ensure it is easily accessible for testing purposes.
    • If you can't drag the link, right-click it and select "Add to bookmarks" or "Bookmark this link."
  2. Testing with Text Spacing Bookmarklet:
    • Navigate to the webpage you want to test.
    • In your bookmarks bar, select the Text Spacing Bookmarklet. This will apply the predefined text spacing to the page. The page should immediately be updated with modified text spacing.
If the bookmarklet doesn't seem to work:
    • Try refreshing the page and applying the bookmarklet again.
    • Check if the page uses iframes, which the bookmarklet may not affect.
  1. Observe Changes:
    • The bookmarklet typically adjusts line height, letter, and paragraph spacing. Carefully observe how the text spacing changes on the page.
  2. Identify and Document Issues:
    • Content Visibility: All content should remain visible and not be cut off.
    • Functionality: All interactive elements (buttons, links, forms) should remain functional.
    • Layout Integrity: Page layout should not break or overlap significantly.
    • Readability: Text should be readable and not overly distorted.
  3. Provide Feedback:
    • Record Findings: Document any issues identified during testing, including screenshots and descriptions.
    • Suggestions for Improvements: Recommend adjusting text spacing to meet accessibility standards and enhance readability.
    • Collaborate: Share your findings and recommendations with the development team to ensure text spacing meets accessibility standards and enhances readability, and necessary adjustments are made.

Scoring Guide

Use this guide to evaluate the web page's performance:

    • Pass: All functionality is usable at 200% and 400% zoom, text increases in size appropriately, and text spacing is readable.
    • Partial Fail: Minor overlapping occurs, but the content is mostly readable and functional.
    • Fail: Text doesn't increase in size, or content overlaps significantly, is cut off, or obscured.
