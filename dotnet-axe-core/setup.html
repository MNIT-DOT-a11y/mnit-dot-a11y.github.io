.NET Axe-Core Selenium Setup

This guide explains how to install Axe-Core Selenium, configure WebDriver, write accessibility test cases, and run dynamic scans against the rendered DOM.

1. Install .NET Tools

Install the .NET SDK and create a new NUnit test project.

dotnet new nunit -n AccessibilityScans
cd AccessibilityScans

2. Install Axe-Core Selenium

Install required NuGet packages:

dotnet add package Deque.AxeCore.Selenium
dotnet add package Deque.AxeCore.Commons
dotnet add package Selenium.WebDriver
dotnet add package Selenium.WebDriver.ChromeDriver

3. Configure Rules

Use WithTags() or DisableRules() to control which WCAG rules are evaluated.

var result = new AxeBuilder(driver)
  .WithTags("wcag2a", "wcag21aa")
  .Analyze();

4. Create Test File

Create a test file such as FullPageScanTests.cs:

using Deque.AxeCore.Selenium;
using NUnit.Framework;
using OpenQA.Selenium;
using OpenQA.Selenium.Chrome;

[TestFixture]
public class FullPageScanTests {
  private IWebDriver driver;

  [SetUp]
  public void SetUp() {
    var options = new ChromeOptions();
    options.AddArgument("--headless");
    driver = new ChromeDriver(options);
  }

  [Test]
  public void HomePage_Should_Have_No_Violations() {
    driver.Navigate().GoToUrl("https://your-app-url.com");
    var result = new AxeBuilder(driver).Analyze();
    Assert.That(result.Violations, Is.Empty);
  }

  [TearDown]
  public void TearDown() {
    driver.Quit();
  }
}

5. Run Scans

Execute tests using:

dotnet test AccessibilityScans.csproj

Violations appear in the test output and can be exported as TRX files for CI/CD pipelines.
