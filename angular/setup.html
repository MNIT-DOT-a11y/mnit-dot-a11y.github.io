Install and Run Cypress Axe-Core with Angular

This guide walks you through cloning the MNIT DOT accessibility training project, running it locally, and executing automated accessibility tests using Cypress and axe-core.

Before You Begin

  • Node.js (LTS recommended)
  • npm (included with Node.js)
  • Angular CLI (npm install -g @angular/cli)
  • Git

Checkpoint: Running node -v and ng version works in your terminal.

1. Clone the Repository

Clone the MNIT DOT A11y repository and navigate to the Angular training project:

git clone https://github.com/mnit-dot-a11y/mnit-dot-a11y.github.io.git
cd mnit-dot-a11y.github.io/angular/accessibility-training

Checkpoint: You are inside the accessibility-training folder.

2. Install Project Dependencies

Install all required Angular dependencies:

npm install

Checkpoint: A node_modules folder is created with no errors.

3. Install Cypress and Axe-Core

Install Cypress and accessibility testing libraries:

npm install --save-dev cypress cypress-axe axe-core

Checkpoint: Cypress appears in package.json under devDependencies.

4. Start the Angular Application

Cypress requires the Angular app to be running locally.

ng serve

Leave this terminal window open.

Checkpoint: Visit http://localhost:4200 and confirm the app loads.

5. Configure Cypress Support File

Open cypress/support/e2e.ts and import the axe commands:

import './commands';
import 'cypress-axe';

Checkpoint: No TypeScript errors appear.

6. Configure TypeScript Types

Update cypress/tsconfig.json:

{
  "compilerOptions": {
    "types": ["cypress", "cypress-axe"]
  }
}

Checkpoint: IntelliSense recognizes cy.checkA11y().

7. Create the Accessibility Test

Create cypress/e2e/accessibility.cy.ts:

describe('Accessibility Audits', () => {
  beforeEach(() => {
    cy.visit('/');
    cy.injectAxe();
  });

  it('Has no detectable accessibility violations on load', () => {
    cy.checkA11y();
  });

  it('Checks main content for serious and critical issues', () => {
    cy.checkA11y('main', {
      includedImpacts: ['serious', 'critical']
    });
  });
});

Checkpoint: The test file appears under cypress/e2e.

8. Run the Tests

Open Cypress in interactive mode:

npx cypress open

Select E2E Testing and run accessibility.cy.ts.

Checkpoint: Tests complete with no accessibility violations.

3. Configure TypeScript Types

To ensure IntelliSense works with custom axe commands, update your cypress/tsconfig.json to include the cypress-axe types:

{
"compilerOptions": {
"types": ["cypress", "cypress-axe"]
}
}

4. Create the Accessibility Test File

Create a new test file at cypress/e2e/accessibility.cy.ts and add the following code to scan your Angular application:

describe('Accessibility Audits', () => {
beforeEach(() => {
cy.visit('/'); // Visit the root of your Angular app
cy.injectAxe(); // Inject the axe-core runtime
});

it('Has no detectable accessibility violations on load', () => {
// Basic check
cy.checkA11y();
});

it('Checks specific element accessibility', () => {
// Check only the main content area with serious/critical impact rules
cy.checkA11y('main', {
includedImpacts: ['serious', 'critical']
});
});
});

5. Run the Tests

You can run your tests in the interactive runner or in headless mode for CI/CD:

# To open the Cypress Test Runner
npx cypress open

# To run tests in the terminal
npx cypress run
