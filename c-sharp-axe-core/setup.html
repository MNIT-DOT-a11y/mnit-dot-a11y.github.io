Instructions to Install and Run Axe-Core Cypress Tests (for C# Teams)

This guide provides step-by-step instructions for setting up and running axe-core accessibility scans using Cypress. While Cypress tests are written in JavaScript, this approach is commonly used by C# development teams as part of CI/CD pipelines, shared accessibility testing strategies, or cross-team quality workflows.

1. Install Node.js

  • Download Node.js: Visit the official Node.js website.
  • Install: Use the LTS version recommended for most users.
  • Verify installation: Run node --version and npm --version.

2. Create a Cypress Project

  • Create a new folder: mkdir axe-cypress-tests
  • Navigate into the folder: cd axe-cypress-tests
  • Initialize npm: npm init -y
  • Install Cypress: npm install cypress --save-dev

3. Add Axe-Core for Cypress

npm install cypress-axe axe-core --save-dev

4. Configure Cypress for Accessibility Testing

Open cypress/support/e2e.js (or support/index.js in older versions) and add the following: 

import 'cypress-axe';

5. Create an Accessibility Test

Create a new test file such as cypress/e2e/accessibility.cy.js and add the following example: 

describe('Accessibility checks', () => {
  it('Runs axe-core on the page', () => {
    cy.visit('https://www.example.com');
    cy.injectAxe();
    cy.checkA11y();
  });
});

6. Run the Tests

  • Open Cypress Test Runner: npx cypress open
  • Run headless (CI-friendly): npx cypress run
  • Review results: Accessibility violations appear in the Cypress output and CI logs.

7. Using Cypress Results in C# Workflows

C# teams commonly integrate Cypress accessibility results by:

  • Running Cypress tests in CI pipelines alongside .NET builds
  • Failing builds when accessibility violations exceed thresholds
  • Parsing Cypress JSON or console output for reporting in C# dashboards
  • Using Cypress for UI accessibility and C# tests for API or backend validation
