Axe-Core Cypress Setup (for C# Teams)

This guide explains how to run axe-core accessibility scans using Cypress. While Cypress tests are written in JavaScript/TypeScript, they are often used by C# teams in CI/CD workflows to test .NET applications.

1. Install Node.js

  • Download Node.js LTS from nodejs.org
  • Verify installation by running node --version and npm --version in your terminal.

2. Create a Cypress Project

Run the following commands to initialize your testing directory:

mkdir axe-cypress-tests
cd axe-cypress-tests
npm init -y
npm install cypress --save-dev

3. Add Axe-Core

Install the necessary accessibility testing plugins:

npm install cypress-axe axe-core --save-dev

4. Configure Cypress

Edit your cypress/support/e2e.js file and add the following import:

import 'cypress-axe';

5. Create Accessibility Test

Create a test file (e.g., cypress/e2e/a11y.cy.js) with the following content:

describe('Accessibility checks', () => {
it('Runs axe-core on the page', () => {
cy.visit('https://www.example.com');
cy.injectAxe();
cy.checkA11y();
});
});

6. Run Tests

  • Interactive Mode: Use npx cypress open to use the GUI runner.
  • Headless Mode: Use npx cypress run for automated CI/CD execution.

7. C# Integration

  • Pipeline Integration: Run Cypress alongside your .NET build steps in Azure DevOps or GitHub Actions.
  • Build Failure: Configure Cypress to exit with a non-zero code on accessibility violations to fail the build.
  • Reporting: Parse Cypress JSON output to display results in C# unit test dashboards.
