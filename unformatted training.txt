Accessibility Training Outline - Draft
Last updated: 10/21/2024
General Guidelines
•	6 sessions over the course of three weeks.
•	Primary Engagement Method: Screen sharing will be used for all demonstrations and explanations.
•	Installation Required: Participants are directed to install tools prior to the beginning of the training.
•	Focus on the GitHub Training Website: The training will include instruction on using the MNIT DOT accessibility checklists and documentation.
•	Hands-on Sessions: There will be at least one hands-on session where participants can complete tasks on their own, with a chance to ask questions.
 
Course 1: Introduction to Accessibility Testing and Tools
Learning Objectives
•	Understand the goals and importance of accessibility testing.
•	Familiarize with the tools conceptually (NVDA, Windows Magnifier, Browser Zoom, Axe).
•	Learn how to navigate the GitHub training website.
Agenda
1.	Welcome and Overview (5 minutes):
–	Introduce training objectives and schedule.
–	Explain session protocols due to recordings (e.g., when to ask questions).
2.	Introduction to Accessibility Testing:
•	Demonstrate how to access and navigate the accessibility checklist.
•	Mention the Setting up Your Environment page.
•	Highlight key sections and explain how to use the checklist during testing, including the How to Test pages.
–	ap of key concepts from the Deque Accessibility Fundamentals course.
–	Emphasize the role of accessibility in quality assurance.
3.	Navigating the GitHub Training Website (25 minutes):
•	Highlight key sections and explain how to use the checklist during testing, including the How to Test pages.
4.	Provide a conceptual overview of:
•	
–	Automated Testing Tools (Axe)
–	NVDA Screen Reader
–	Windows Magnifier
–	Browser Zoom Functions
–	Explain the importance of each tool in accessibility testing.
 
Course 2: Keyboard Accessibility Testing
Learning Objectives
•	Understand the principles and importance of keyboard accessibility.
•	Learn how to perform keyboard accessibility testing using the GitHub checklist.
•	Participate in a practical breakout session using keyboard navigation.
Agenda
1.	Recap and Introduction:
–	Review key points from Day 1.
–	Introduce keyboard accessibility testing.
2.	Principles of Keyboard Accessibility (15 minutes):
–	Presentation:
•	Explain why keyboard accessibility is critical.
•	Discuss common issues and best practices.
3.	Demonstration of Keyboard Navigation (20 minutes):
–	Screen Sharing:
•	Navigate a sample application using only the keyboard.
•	Identify common issues and demonstrate how to document them using the GitHub checklist.
•	Emphasize elements like interactive components and proper focus management.
4.	Hands-on session: Hands-on Keyboard Navigation (15 minutes):
–	Instructions:
•	Participants will be provided with a link to a test website.
•	Spend 10 minutes navigating the website using only the keyboard.
•	Block 5 minutes for technical issues and questions.
•	Encourage participants to note any accessibility issues they encounter.
–	Note:
•	This is the only time participants must navigate to an external website during the training.
 
Course 3: Magnification and Low-Vision Accessibility Testing
Learning Objectives
•	Understand challenges faced by users with low vision.
•	Learn how to perform magnification testing using Windows Magnifier and Browser Zoom.
•	Identify issues related to magnification using the GitHub checklist.
Agenda
1.	Recap and Introduction (5 minutes):
–	Summarize insights from Day 2.
–	Introduce magnification and its role in accessibility testing.
2.	Understanding Low Vision Challenges (10 minutes):
–	Presentation:
•	Discuss types of visual impairments and their impact on users.
•	Explain how design choices affect readability and usability.
3.	Demonstration of Magnification Testing (40 minutes):
–	Screen Sharing:
•	Windows Magnifier:
–	Show how to use Windows Magnifier at various zoom levels (e.g., 200-400%).
–	Demonstrate testing on a sample website.
•	Browser Zoom:
–	Explain at a high level the difference between Windows Magnifier and browser Zoom.
–	Demonstrate how to adjust zoom settings in common browsers (e.g., Chrome, Firefox).
–	Highlight potential issues like content clipping, overlapping elements, and loss of functionality.
•	Identify issues using the GitHub checklist.
 
Course 4: Screen Reader Accessibility Testing with NVDA and Automated Testing with Axe
Learning Objectives
•	Understand the basics of screen reader accessibility.
•	Learn how to perform screen reader testing using NVDA.
•	Gain an understanding of automated testing with Axe.
•	Identify common issues using the GitHub accessibility checklist.
Agenda
1.	Recap and Introduction:
–	Review key points from Day 3.
–	Introduce screen reader accessibility testing.
2.	Basics of Screen Reader Accessibility (10 minutes):
–	Presentation:
•	Explain how screen readers assist users.
•	Discuss common barriers like missing alt text and improper headings.
3.	Demonstration of Screen Reader Testing with NVDA (25 minutes):
–	Screen Sharing:
•	Use NVDA to navigate a sample website (pick one of the inaccessible demos).
•	Demonstrate interaction with different elements (e.g., menus, forms).
•	Identify issues using the GitHub website.
4.	Introduction to Automated Testing with Axe (15 minutes):
–	Screen Sharing:
•	Demonstrate how to run Axe on a sample web page.
•	Explain how to interpret Axe reports.
•	Discuss how Axe complements manual testing.
 
Course 5: Integrating Accessibility Testing into QA Processes
Learning Objectives
•	Learn how to incorporate accessibility testing into daily QA workflows.
•	Understand effective documentation and reporting of accessibility issues.
•	Access resources for continued learning and support.
Agenda
1.	Recap and Introduction (5 minutes):
–	Summarize learnings from previous days.
–	Highlight the importance of ongoing accessibility efforts.
2.	Integrating Accessibility into QA Workflows (15 minutes):
–	Presentation:
•	Strategies for including accessibility checks in testing cycles.
•	Discuss how to prioritize and report accessibility issues.
3.	Documentation and Reporting (20 minutes):
–	Screen Sharing:
•	Demonstrate using the scoping template, reporting spreadsheet and executive summary template for reporting accessibility issues.
•	Show examples of well-documented issues (provide links so they can read them later).
•	Emphasize the role of accurate reporting in improving accessibility.
4.	Resources for Continued Learning:
–	Share a curated list of tools, guidelines, and communities.
•	Discuss future MNIT office hours.
–	Encourage participation in accessibility communities.
5.	Closing Remarks:
–	Address final questions.
–	Recap training objectives and outline the next steps.
