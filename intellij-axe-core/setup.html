IntelliJ + Axe Linter

This guide explains how to install IntelliJ IDEA, configure the Axe Linter plugin, set up rule configuration using axe-linter.yml, and run static accessibility scans directly in the editor.

Download Sample IntelliJ Project

Install IntelliJ

Install IntelliJ IDEA Community or Ultimate and ensure you have JDK 17+ installed.

java -version

Install Axe Linter

You may install either the free Accessibility Linter or the licensed Deque Axe DevTools Linter.

Free Accessibility Linter (Marketplace)

  • Open IntelliJ → Settings → Plugins
  • Select Marketplace
  • Search for Accessibility Linter
  • Click Install and restart IntelliJ

Deque Axe DevTools Linter (Licensed)

  • Download the plugin from your Deque Agora account
  • Open IntelliJ → Settings → Plugins
  • Click the gear icon → Install Plugin from Disk
  • Select the .jar file and restart

Configure Rules

Create an axe-linter.yml file in the project root to control which rules are evaluated.

tags:
  - wcag2a
  - wcag21a
  - wcag2aa
  - wcag21aa

rules:
  html-has-lang: false

Create Test File

Open any HTML, JSX, or template file. Axe Linter will automatically highlight violations.

<!-- Example with violations -->
<html>
  <body>
    <h1></h1>
    <img src="photo.jpg">
    <input type="text">
  </body>
</html>

Run Scans

Violations appear as wavy underlines and in the Problems panel. Hover to view WCAG mappings and remediation guidance.
