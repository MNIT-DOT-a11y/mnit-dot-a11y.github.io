Instructions to Install and Run Axe-Core Selenium scripts with Python

This guide provides step-by-step instructions for setting up your environment and running an accessibility scan on a web page using Python, Axe-core, and Selenium.

1. Install Python and Visual Studio Code

  • Install Python: Download the latest stable build from the official Python website. When running the installer, be sure to check the box to "Add python.exe to PATH" to ensure you can run Python and pip commands from any terminal.
  • Install VS Code: Visual Studio Code should already be installed.

2. Set up a Python Project in VS Code

  • Create Project Folder: Open your terminal (e.g., PowerShell) and run the following commands to create and enter a project directory: 
    mkdir python-axe-selenium
cd python-axe-selenium
  • Open the Project in VS Code: Open the new folder in Visual Studio Code by running the following command: 
    code .
  • Create a Virtual Environment: In VS Code, open the integrated terminal (Ctrl+`). Create a virtual environment using the command below. 
    python -m venv .venv

    VS Code will likely prompt you to select this new environment. If not, use the Command Palette (Ctrl+Shift+P) and type "Python: Select Interpreter" to choose the `.venv` interpreter.

3. Install Required Libraries

  • Activate Virtual Environment: In the VS Code terminal, activate your environment by running this command: 
    .venv\Scripts\activate
  • Install Packages: Use `pip` to install the necessary libraries for your tests: 
    pip install selenium pytest axe-selenium-python webdriver-manager pytest-html

4. Download and Install the cffi Wheel File

For certain dependencies, it may be necessary to install a specific wheel file manually. The filename `cffi-1.17.1-cp313-cp313-win_amd64.whl` indicates it's built for Python 3.13 on a 64-bit Windows system.

  • Download the Wheel File: You will need to download this file from a trusted source, such as the cffi project page on PyPI: 
    https://files.pythonhosted.org/packages/b9/7d/13e9a98e826b6f793f6d3d0e34fb8/cffi-1.17.1-cp313-cp313-win_amd64.whl
  • Install the `.whl` file: Once the file is downloaded, navigate to its directory in your terminal and run the following command. Be sure to replace the filename if you have a different version. 
    pip install cffi-1.17.1-cp313-cp313-win_amd64.whl

5. Create the Python Test File

  • Create the File: In VS Code's File Explorer, create a new file named `test_accessibility.py` in the root of your project folder.
  • Add the Code: Copy and paste the corrected Python code into this new file. The code uses the `parametrize` marker to test multiple sites and generates a JSON report for any violations. 
    import pytest
import os
import json
from selenium import webdriver
from axe_selenium_python import Axe
from webdriver_manager.chrome import ChromeDriverManager
from selenium.webdriver.chrome.service import Service

# The list of sites to test for accessibility.
SITES_TO_TEST = [
    "https://mn.gov/mnit/about-mnit/accessibility/office-of-inaccessibility.jsp",
    "https://www.google.com",
    "https://www.bing.com",
    "https://www.yahoo.com"
]

@pytest.fixture(scope="function")
def driver():
    """Sets up a fresh WebDriver instance for each test function."""
    service = Service(ChromeDriverManager().install())
    options = webdriver.ChromeOptions()
    options.add_argument("--start-maximized")
    web_driver = webdriver.Chrome(service=service, options=options)
    yield web_driver
    web_driver.quit()

def create_report_directories():
    """Ensures the directory for storing reports exists."""
    os.makedirs("target/a11y-json-reports", exist_ok=True)

def generate_json_report(url: str, results: dict):
    """
    Generates a JSON report from the axe-core scan results.
    The filename is based on the URL to ensure it's unique.
    """
    create_report_directories()
    filename = url.replace("https://", "").replace("http://", "").replace("/", "_").replace(".", "_") + "_axe_results.json"
    filepath = os.path.join("target/a11y-json-reports", filename)
    with open(filepath, "w") as f:
        json.dump(results, f, indent=4)
    print(f"Axe-core JSON results saved to: {os.path.abspath(filepath)}")

@pytest.mark.parametrize("site_url", SITES_TO_TEST)
def test_accessibility_on_sites(driver, site_url):
    """
    This test runs an axe-core accessibility scan on each URL in SITES_TO_TEST.
    It generates a JSON report and fails the test if any violations are found.
    """
    print(f"\nNavigating to page: {site_url}")
    driver.get(site_url)
    print(f"Executing axe-core accessibility scan on {site_url}...")

    # Run the axe-core analysis.
    axe = Axe(driver)
    results = axe.run()

    # Generate a JSON report from the results.
    generate_json_report(site_url, results)

    # Check for violations and handle them.
    violations = results.get('violations', [])
    if violations:
        print(f"--- Accessibility Violations Found on {site_url} ---")
        for violation in violations:
            print(f"Rule ID: {violation['id']}")
            print(f"Description: {violation['description']}")
            print(f"Help URL: {violation['helpUrl']}")
            print(f"Impact: {violation['impact']}")
            print("Nodes:")
            for node in violation.get('nodes', []):
                print(f"     HTML: {node['html']}")
                print(f"     Target: {node['target']}")
                print(f"     Failure Summary: {node['failureSummary']}")
                print("-----")

        # If violations exist, fail the test and provide a message.
        pytest.fail(f"Accessibility violations found on {site_url}. Check the generated JSON report for details.")
    else:
        # If no violations, print a success message.
        print(f"No accessibility violations found on {site_url}!")

6. Run the Tests and View Results

Run the Tests: In the VS Code terminal, run the following command to execute your tests and generate a detailed HTML report: 

pytest --html=test_report.html

Check Test Results:

  • Terminal Output: Test results will be displayed directly in the VS Code terminal.
  • JSON Reports: Detailed accessibility results for each site are saved as separate JSON files in the `target/a11y-json-reports` folder.
  • HTML Report: The overall test summary is saved in `test_report.html` in your project's root directory. You can open this file in any web browser to view a comprehensive report of the test run.
